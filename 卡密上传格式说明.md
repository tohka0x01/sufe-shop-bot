# 卡密上传格式说明

## 支持的文件格式
- **文件类型**：仅支持纯文本文件（.txt）
- **文件编码**：UTF-8（支持中文和特殊字符）
- **文件大小**：建议不超过 10MB

## 卡密格式规则

### 1. 单行卡密
每行一个卡密，直接换行即可：
```
CODE001
CODE002
CODE003
```

### 2. 多行卡密
多行卡密用空行分隔，例如：
```
租户名: tenant001
用户名: user@email.com
密码: password123
（空行）
租户名: tenant002
用户名: user2@email.com
密码: password456
```

### 3. 混合格式
可以在同一个文件中混合使用单行和多行卡密：
```
SINGLE-CODE-001
SINGLE-CODE-002
（空行）
账号: multi001
密码: pass123
有效期: 2025-12-31
（空行）
SINGLE-CODE-003
```

## 分隔符说明
- **空行**：用于分隔不同的卡密
- **分隔线**：`===` 或 `---` 会被识别为空行，可用于视觉分隔
- **注意**：多行卡密内部不要有空行，否则会被识别为两个不同的卡密

## 注意事项
1. 上传前请检查文件格式是否正确
2. 建议先用少量卡密测试上传功能
3. 上传后请在后台查看卡密是否正确识别
4. 如果卡密包含特殊字符，请确保文件编码为 UTF-8
5. 不要在卡密中包含 HTML 或 Markdown 标记

## 特殊字符支持
支持以下特殊字符：
- 基础符号：@ # $ % & * ( ) - _ + = [ ] { } : ; " ' , . / \ | ? ! ~
- 中文字符：全角和半角
- 数字：0-9
- 英文：a-z, A-Z
- 空格和制表符

## 故障排除
1. **上传失败**：检查文件是否为 .txt 格式
2. **卡密识别错误**：检查是否有多余的空行或特殊字符
3. **中文乱码**：确保文件保存为 UTF-8 编码
4. **卡密丢失**：检查多行卡密之间是否正确使用空行分隔
