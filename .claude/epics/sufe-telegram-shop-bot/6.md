# Task: CSS架构统一

## Description
统一分散在多个位置的CSS定义，建立规范的CSS架构。将现有的三个CSS位置（内联样式、模板中的style标签、分散的CSS文件）合并到统一的文件结构中。创建CSS变量系统，为后续的主题系统实现打下基础。

## Acceptance Criteria
- [ ] 创建/static/css/theme.css作为主样式文件
- [ ] 建立CSS变量系统，包含颜色、间距、字体等设计标记
- [ ] 移除所有内联样式到外部CSS文件
- [ ] 移除模板中的<style>标签，改为引用外部CSS
- [ ] CSS文件结构清晰，有合理的模块化组织
- [ ] 所有页面样式保持不变或有所改进

## Technical Details
- 实现方法：
  - 审计所有HTML模板中的内联样式和style标签
  - 创建CSS变量定义文件（variables.css）
  - 按功能模块组织CSS（layout.css, components.css, utilities.css）
  - 使用CSS自定义属性实现变量系统
- 关键考虑：
  - 保持向后兼容性
  - 确保CSS加载顺序正确
  - 优化CSS文件大小
  - 支持未来的主题切换
- 受影响的文件：
  - templates/*.html
  - static/css/（新建目录）
  - templates/base.html（更新CSS引用）

## Dependencies
- [ ] 需要完整审计现有的样式定义
- [ ] 确保静态文件服务配置正确
